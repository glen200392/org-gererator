# Contributing

Thanks for your interest in improving this project.

## Development Model

This is a single-file web app for easy deployment and maintenance.

## How to Contribute

1. Fork the repository.
2. Create a feature branch.
3. Keep changes focused and small.
4. Validate core workflows manually:
   - sample loading
   - Excel upload
   - PNG/PDF/PPTX export
5. Open a pull request with a clear summary.

## Coding Guidelines

- Preserve existing behavior unless the change request requires otherwise.
- Prefer readable, minimal JavaScript changes.
- Do not add heavy frameworks for small features.

## Pull Request Checklist

- [ ] Scope is clear and limited.
- [ ] Manual smoke test completed.
- [ ] No unrelated formatting churn.
- [ ] Changelog updated when behavior changes.
